概率论：最大似然、MAP 与交叉熵

2026-07-10 数学 概率论 概率论 最大似然 MAP 交叉熵 机器学习

训练模型到底在估计什么

很多机器学习模型都可以这样看：我们假设数据来自某个带参数的概率模型，然后通过训练数据估计参数。

参数可以是线性回归里的权重，也可以是神经网络里的全部参数。训练的核心问题是：什么样的参数最能解释我们看到的数据？

最大似然就是最常见的回答。

似然是什么

假设模型参数是 theta，数据是 D。

概率 P(D | theta) 表示：如果参数是 theta，观察到这批数据的概率有多大。

当我们把数据看成已知，把 theta 看成变量时，这个量就叫似然：

L(theta) = P(D | theta)

最大似然估计就是选择让数据最可能出现的参数：

theta_hat = argmax_theta P(D | theta)

为什么取对数

如果数据样本独立同分布：

P(D | theta) = product_i P(x_i | theta)

很多小概率相乘会非常小，数值上不稳定。取对数后，乘法变加法：

log P(D | theta) = sum_i log P(x_i | theta)

所以训练时通常最大化 log likelihood，或者等价地最小化 negative log likelihood：

NLL = - sum_i log P(x_i | theta)

这就是“负对数概率”在机器学习里反复出现的原因。

分类与交叉熵

分类模型输出的是类别分布 q(y | x)。真实标签如果是 one-hot 分布 p(y | x)，训练目标常写成交叉熵：

H(p, q) = - sum_y p(y | x) log q(y | x)

因为 p 是 one-hot，只有正确类别那一项保留下来：

loss = - log q(y_true | x)

这正是负对数似然。

所以分类交叉熵不是凭空发明的损失函数，它就是在最大化模型给正确标签分配的概率。

语言模型里的最大似然

语言模型训练目标也可以写成最大似然。

一句 token 序列：

x1, x2, ..., xT

自回归模型会分解为：

P(x1, ..., xT) = product_t P(x_t | x_<t)

训练时最小化：

- sum_t log P(x_t | x_<t)

这和分类交叉熵形式一样，只是类别空间变成了词表，每个位置都要预测下一个 token。

MAP：把先验也放进来

最大似然只看数据对参数的支持程度。最大后验估计，也就是 MAP，会进一步考虑参数先验：

theta_hat = argmax_theta P(theta | D)

根据贝叶斯公式：

P(theta | D) proportional to P(D | theta) P(theta)

所以 MAP 等价于同时考虑似然和先验。

如果先验偏好较小的参数，最后就会出现类似正则化的效果。

正则化的概率解释

线性模型里常见的 L2 正则：

loss = data_loss + lambda * ||w||_2^2

可以从 MAP 角度理解为：我们给参数 w 一个以 0 为中心的高斯先验，希望权重不要太大。

L1 正则也有概率解释，常和拉普拉斯先验联系起来。它更鼓励稀疏参数。

这个视角很重要：正则化不只是工程技巧，也可以看成对参数的先验偏好。

小结

最大似然回答“什么参数最能解释数据”，负对数似然把最大化概率变成最小化损失，交叉熵是分类和语言模型中最常见的负对数似然形式，MAP 则进一步把先验加入参数估计。

理解这条线，就能把概率论、损失函数和模型训练连起来。
