概率论：条件概率、贝叶斯公式与朴素贝叶斯

2026-07-10 数学 概率论 概率论 条件概率 贝叶斯 朴素贝叶斯 机器学习

为什么先学条件概率

机器学习里几乎所有“看到输入以后再判断”的问题，都可以写成条件概率。

分类不是只问 P(y)，而是问 P(y | x)。语言模型不是只问某个 token 的概率，而是问 P(next token | context)。强化学习里的策略也常写成 pi(a | s)，表示在状态 s 下选择动作 a 的概率。

条件概率让模型从“总体上可能发生什么”，转向“在已经观察到一些信息后，什么更可能发生”。

条件概率

条件概率的定义是：

P(A | B) = P(A and B) / P(B)

这里要求 P(B) > 0。

直觉是：既然已经知道 B 发生了，样本空间就缩小到 B 内部。我们只关心在这个更小的世界里，A 发生的比例。

在机器学习里，B 经常是观测到的特征，A 经常是要预测的标签。

乘法公式

把条件概率公式移项，可以得到：

P(A and B) = P(A | B) P(B)

这条式子非常基础，但很有用。它告诉我们，联合概率可以被拆成“先发生一个事件，再在这个条件下发生另一个事件”。

如果扩展到多个变量，就会得到链式分解：

P(x1, x2, x3) = P(x1) P(x2 | x1) P(x3 | x1, x2)

自回归语言模型正是这种思想的深度学习版本：一句话的概率被拆成一连串下一个 token 的条件概率。

贝叶斯公式

贝叶斯公式把 P(A | B) 和 P(B | A) 联系起来：

P(A | B) = P(B | A) P(A) / P(B)

在机器学习里，经常写成：

P(y | x) = P(x | y) P(y) / P(x)

这几个量的含义很清楚：

符号 含义
P(y) 先验概率，还没看输入时某个类别本来有多常见
`P(x y)`
`P(y x)`
P(x) 证据项，负责归一化

预测分类时，P(x) 对所有类别一样，所以常比较：

P(x | y) P(y)

这就是很多生成式分类模型的基本思路。

朴素贝叶斯为什么朴素

朴素贝叶斯的关键假设是：给定类别后，各个特征条件独立。

P(x1, x2, ..., xn | y) = P(x1 | y) P(x2 | y) ... P(xn | y)

这个假设经常不完全真实。文本里的词明显会互相影响，图像里的像素也不是独立的。

但它带来的好处很大：原本很难估计的高维联合概率，被拆成很多低维概率。样本不多时，这种简化反而常常很稳。

文本分类里的直觉

假设要判断一封邮件是不是垃圾邮件。

朴素贝叶斯会比较：

P(spam | words) 和 P(normal | words)

根据贝叶斯公式，可以转成比较：

P(spam) * product_i P(word_i | spam)
P(normal) * product_i P(word_i | normal)

如果“中奖”“免费”“点击”这类词在垃圾邮件中更常见，它们会提高 P(spam | words)。

实际计算时通常取对数，避免很多小概率相乘造成数值下溢：

log P(y) + sum_i log P(word_i | y)

和现代模型的关系

朴素贝叶斯很简单，但它教会我们三件事：

  1. 预测可以看成后验概率估计。
  2. 强假设可以降低估计难度。
  3. 对数概率比原始概率更适合计算。

这些思想在现代模型里仍然存在。语言模型训练的是条件概率，交叉熵本质上在惩罚正确 token 的负对数概率，很多对齐算法也会直接比较对数概率差。

小结

条件概率描述“已知信息后的不确定性”，贝叶斯公式把先验、似然和后验连起来，朴素贝叶斯则用条件独立假设把高维概率拆成可估计的小块。

理解这一条线，再看分类、生成、语言模型和强化学习策略，会顺很多。
