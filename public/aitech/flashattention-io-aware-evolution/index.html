FlashAttention 详解：Online Softmax、I/O 优化与 v1-v4 演进

2026-07-10 AiTech LLM 推理优化 FlashAttention Online Softmax CUDA Hopper Blackwell Flash-Decoding

标准 attention 的公式只有一行：

\[ O=\operatorname{softmax}\left(\frac{QK^\top}{\sqrt d}\right)V. \]

但当序列很长时，中间分数矩阵 \(S=QK^\top\) 有 \(N^2\) 个元素。朴素实现会把它写入 GPU 高带宽显存（HBM），再读回来做 softmax，再写入概率矩阵，最后又读回来乘 \(V\)。很多时间并没有花在乘加本身，而是花在不同存储层级之间搬运数据。

FlashAttention 的突破不是近似 attention，也不是把平方 FLOPs 变成线性 FLOPs。它在不物化完整 \(N\times N\) 矩阵的前提下，分块计算完全相同的 softmax attention，用更多片上复用和必要的重计算换取更少 HBM I/O。

先给出核心结论

  1. FlashAttention 是精确算法：在浮点舍入允许的范围内，它与标准 softmax attention 计算同一结果。
  2. 它的算术复杂度仍为 \(O(N^2d)\)，主要降低的是辅助显存和 HBM 读写。
  3. 能够分块的关键是 online softmax：每次看到新的一块分数，都能用新的最大值正确缩放旧累积量。
  4. FA2 改进 GPU 工作划分；FA3 针对 Hopper 的 WGMMA/TMA 与异步流水；FA4 针对 Blackwell 的 TMEM、2-CTA MMA 和新的 softmax 实现。
  5. Flash-Decoding 是针对单 token decode 并行度不足的变体，不是 FlashAttention 的下一代版本。
  6. flash-attn 包版本、论文中的 FA1-FA4、PyTorch SDPA backend 是三套不同概念。

GPU 为什么不只看 FLOPs

现代 GPU 大致有多级存储：

层级 容量 速度/带宽 典型用途
registers 最小 最快 单线程局部数据
shared memory / SRAM 较小 很快 线程块协作、tile 复用
L2 cache 中等 较快 SM 之间缓存
HBM 最大 相对慢 模型权重、激活、KV Cache

Tensor Core 的矩阵乘吞吐增长很快，但只有数据及时送到计算单元才能发挥。若一个 kernel 频繁读写 HBM，性能上限由内存带宽而不是峰值 FLOPs 决定。

这可以用 roofline 思路理解。算术强度定义为：

\[ I=\frac{\text{FLOPs}}{\text{bytes moved}}. \]

减少相同计算所需的数据搬运，会提高算术强度，让程序更接近计算吞吐上限。FlashAttention 正是在 attention 的数据流上做这件事。

朴素 attention 到底物化了什么

设单头：

Q, K, V: [N, d]
S = Q K^T: [N, N]
P = softmax(S): [N, N]
O = P V: [N, d]

即使 \(Q,K,V,O\) 都只占 \(O(Nd)\)，分数和概率矩阵仍占 \(O(N^2)\)。以 FP16、单头、\(N=16384\) 为例，一个完整 \(N^2\) 矩阵约 512 MiB；多头和反向传播会进一步放大压力。

一种初步优化是 kernel fusion：计算分数后立即做 softmax，减少中间写回。但 SRAM 装不下整个长序列的分数矩阵，仍需要分块；而 softmax 的分母依赖一整行所有元素，分块后必须回答“后面出现更大值时，前面累积的指数和怎么办”。

答案就是 online softmax。

从稳定 softmax 推导 online softmax

对一行分数 \(x_1,\ldots,x_N\)，稳定 softmax 使用行最大值：

\[ m=\max_i x_i, \qquad \operatorname{softmax}(x_i)=\frac{e^{x_i-m}}{\sum_j e^{x_j-m}}. \]

定义归一化因子：

\[ \ell=\sum_j e^{x_j-m}. \]

假设已经处理旧分块，保存旧最大值 \(m_{old}\)、旧指数和 \(\ell_{old}\)，以及未归一化输出累积量：

\[ u_{old}=\sum_{j\in old}e^{x_j-m_{old}}v_j. \]

新分块的最大值为 \(m_b\)，合并后的最大值是：

\[ m_{new}=\max(m_{old},m_b). \]

旧累积量原本以 \(m_{old}\) 为指数基准。换成 \(m_{new}\) 后，需要乘：

\[ \alpha=e^{m_{old}-m_{new}}. \]

于是可以在线更新：

\[ \ell_{new} =\alpha\ell_{old} +\sum_{j\in block}e^{x_j-m_{new}}, \]
\[ u_{new} =\alpha u_{old} +\sum_{j\in block}e^{x_j-m_{new}}v_j. \]

遍历全部 Key/Value 分块后：

\[ o=\frac{u}{\ell}. \]

这套递推是精确恒等变换，不要求先知道全局最大值，也不用保存全部分数。

FlashAttention 前向过程

把 Query 行切成 \(B_r\) 大小的块，把 Key/Value 列切成 \(B_c\) 大小的块。对每个 Query 块：

  1. 把 \(Q_i\) 载入片上存储；
  2. 初始化每行的 \(m_i=-\infty\)、\(\ell_i=0\) 和输出累积量；
  3. 逐个加载 \(K_j,V_j\) tile；
  4. 在片上计算 \(S_{ij}=Q_iK_j^\top/\sqrt d\)；
  5. 应用 causal/window mask；
  6. 用 online softmax 更新 \(m_i,\ell_i,O_i\)；
  7. 完成所有 KV tile 后，只把最终 \(O_i\) 和必要统计量写回 HBM。

这里需要理解两种循环顺序的取舍。以 Query tile 为外层可以让 Query 和输出长时间驻留片上，但会重复读取 K/V；以 KV tile 为外层可以复用 K/V，却需要维护更多输出 tile。具体实现会结合 SRAM 容量、head dimension 和 GPU 架构选择。

causal mask 如何工作

因果 attention 只允许位置 \(i\) 读取 \(j\le i\)。分块后：

  • 完全位于对角线左侧的 tile 全部有效；
  • 完全位于右侧的 tile 可以跳过；
  • 与对角线相交的 tile 在片上应用三角 mask。

因此 causal kernel 不必计算完整方阵，理想算术工作量也接近非因果的一半。

反向传播为什么使用重计算

传统反向传播会保存 \(P=\operatorname{softmax}(S)\)，以便计算 \(dV,dQ,dK\)。这又引入平方级激活内存。

FlashAttention 前向只保存输出和每行 log-sum-exp 等线性大小统计量。反向时重新加载 Q/K/V tile，重算局部分数和概率，再累积梯度。

这看起来“做了更多计算”，却可能更快，因为：

  • 重算使用高吞吐矩阵乘；
  • 避免保存和读取巨大的概率矩阵；
  • 降低峰值显存后可使用更大 batch 或更长序列。

这是典型的 compute-for-memory tradeoff：在现代 GPU 上，便宜重算可能优于昂贵搬运。

FlashAttention-1：确立 I/O-aware 路线

FlashAttention-1 在 2022 年系统化提出 IO-aware exact attention：

  • 用 tiling 让 Q/K/V tile 在 SRAM 中参与 fused softmax 与矩阵乘；
  • 不把完整注意力矩阵写入 HBM；
  • 反向通过重计算避免保存平方级激活；
  • 给出不同 SRAM 大小下的 HBM I/O 分析。

它证明了一个重要观点：只比较 FLOPs 会错过 attention 的主要系统瓶颈。算法设计必须把存储层次纳入复杂度模型。

FA1 仍有 GPU 利用率问题。尤其在 head 数和 batch 较小时，线程块数量有限；warp 之间对输出的工作划分也会产生 shared-memory 通信和同步。

FlashAttention-2：更好的并行与工作划分

FlashAttention-2 保留相同数学核心，重点减少非矩阵乘操作并提高 GPU 占用。

减少非 matmul FLOPs

Tensor Core 的矩阵乘极快，而标量指数、比较、边界检查和类型转换相对昂贵。FA2 调整 online softmax 的更新方式，尽量减少 rescale 与非矩阵乘工作，让更多时间落在高吞吐 GEMM 上。

增加序列维度并行

FA1 主要在 batch 和 head 上分配线程块。当 batch * heads 较小时，GPU 上可并行的 block 不够。FA2 还能沿 Query 序列维度划分线程块，因此即使 batch 小、序列长，也有足够任务占满 SM。

改进 warp 分工

FA1 的一种分工让不同 warp 各持有 K/V 的一部分，并共同更新同一 Query 输出，需要在 shared memory 中归约。FA2 更倾向让 warp 共享 K/V tile、各自负责不同 Query 行，减少 warp 间通信。

论文在 A100 的特定形状下报告相对 FA1 约 2 倍加速，并达到理论峰值的 50%-73%。这不是所有形状的固定倍数，但说明工作划分和占用率与核心算法同样重要。

Flash-Decoding：为什么单 token decode 需要另一种切法

训练和 prefill 有很多 Query，可以沿 Query 序列分块并行。自回归 decode 通常每条序列只有一个新 Query：

Q: [batch, heads, 1, d]
K,V: [batch, kv_heads, context_len, d]

即使上下文很长，Query 维度也没有足够 tile。普通 FlashAttention 可能只能为每个 batch/head 启动少量线程块，GPU 占用不足。

Flash-Decoding 沿 KV 序列长度再切一维：

  1. 多个线程块并行处理不同 KV chunk；
  2. 每个 chunk 计算局部输出、最大值和指数和；
  3. 第二步用 log-sum-exp 统计量精确合并所有 chunk。

若第 \(j\) 个 chunk 得到归一化局部输出 \(o_j\)、局部最大值 \(m_j\) 和局部分母 \(\ell_j\)，令：

\[ m=\max_j m_j, \qquad \ell=\sum_j e^{m_j-m}\ell_j, \]

则全局输出为：

\[ o=\frac{\sum_j e^{m_j-m}\ell_j o_j}{\ell}. \]

这仍然是精确 softmax。官方博客在极长序列的特定 benchmark 中报告最高约 8 倍端到端提升；收益会随 batch、head 数、上下文长度和 baseline 改变。Flash-Decoding 从 flash-attn 2.2 起进入实现，但它应理解为 decode kernel 路线，而不是“FA3”。

FlashAttention-3：为 Hopper 重排流水线

FlashAttention-3 面向 NVIDIA Hopper（H100/H800）。Hopper 提供两项关键能力：

  • WGMMA：warpgroup 级异步矩阵乘；
  • TMA：Tensor Memory Accelerator，异步搬运多维 tile。

如果仍按上一代同步流程编程，新硬件不会自动变快。FA3 设计了更深的异步流水。

warp specialization

不同 warpgroup 承担不同角色：一部分负责 TMA 搬运 Q/K/V，一部分执行 WGMMA，另一部分处理 softmax。生产者和消费者通过 barrier 协调，数据搬运与计算可以重叠。

GEMM 与 softmax 重叠

Attention tile 交替执行两类工作：Tensor Core 上的矩阵乘和 special function unit/普通 ALU 上的指数、最大值与归一化。FA3 使用 ping-pong 或 intra-warpgroup pipeline，在一块分数做 softmax 时启动下一块 GEMM，减少功能单元互相等待。

FP8 与误差控制

Hopper 的 FP8 Tensor Core 吞吐很高，但 Q/K 的动态范围和量化误差会影响 attention。FA3 使用 block quantization，并通过 incoherent processing 等技巧降低异常值影响。

论文在 H100 的测试中报告 FP16 最高约 740 TFLOPS、约 75% 理论峰值，FP8 接近 1.2 PFLOPS，并在部分设置下相对 FA2 提升 1.5-2 倍。它高度依赖 Hopper 特性，不能把同样优化直接套在 Ampere 上。

FlashAttention-4：Blackwell 上 softmax 成了更明显的瓶颈

FlashAttention-4 于 2026 年提出，目标硬件是 NVIDIA Blackwell B200/GB200。Blackwell 的 Tensor Core 吞吐继续大幅增长，但 shared memory 带宽和指数等特殊函数吞吐没有同比例增长。结果是：矩阵乘更快后，softmax 与数据移动占比反而更高。

FA4 的主要方向包括：

全异步 MMA 与更大 tile

利用 Blackwell 的异步 MMA、更大的 tile 和 Tensor Memory（TMEM）保存矩阵乘累积结果，让寄存器与 shared memory 压力重新平衡。

2-CTA MMA

两个 cooperative thread arrays 协同执行矩阵乘，改善大 tile 的数据共享与吞吐。这要求更精细的同步和资源分配。

软件模拟指数

当硬件 exponential unit 成为瓶颈时，FA4 使用基于 exp2/FMA 的近似与条件 rescaling，把部分 softmax 工作转移到吞吐更充足的普通算术管线。这里的“精确 attention”指算法目标仍是 softmax attention；任何具体实现都受浮点近似和舍入影响，与常见快速数学函数一样需要误差测试。

CuTe DSL

FA4 使用 Python 中的 CuTe DSL 表达 kernel，而不是主要依赖巨大的 C++ template 实例化。论文报告其测试中编译时间相对传统模板方案降低约 20-30 倍，同时便于快速探索硬件调度。

FA4 论文在 B200 的特定 BF16 benchmark 中报告最高约 1613 TFLOPS、约 71% 峰值，并相对 cuDNN 9.13 最高约 1.3 倍、相对 Triton baseline 最高约 2.7 倍。到 2026 年 7 月，官方仓库中的 FA4 发布仍带 beta/pre-release 标记；“论文已经发布”和“所有形状都已生产稳定”不能画等号。

四代演进放在一张表里

版本 主要硬件 核心问题 代表性改进
FA1 Ampere 等 避免物化 N×N attention tiling、online softmax、反向重计算
FA2 Ampere/Ada 提高占用率，减少非 matmul 工作 序列维度并行、新 warp 分工、更少 rescale
FA3 Hopper 利用异步硬件并重叠计算 WGMMA、TMA、warp specialization、FP8
FA4 Blackwell Tensor Core 与 softmax/存储扩展不对称 TMEM、2-CTA MMA、软件指数、CuTe DSL

这条时间线体现了一个规律：数学公式基本不变，优化重点随硬件瓶颈移动。FA1 解决 HBM I/O，FA2 解决工作划分，FA3 解决异步流水，FA4 进一步处理 softmax 和片上资源相对落后的问题。

它与其他概念有什么区别

FlashAttention 与线性注意力

FlashAttention 保留全部 Query-Key 交互，核心 FLOPs 仍为平方级；线性注意力通过核分解、递推状态或其他结构改变算法，试图降低随序列长度增长的计算。前者 exact 且易替换，后者可能具有更低渐近复杂度但改变模型归纳偏置。

FlashAttention 与 PyTorch SDPA

torch.nn.functional.scaled_dot_product_attention 是统一 API/dispatcher。根据设备、dtype、形状、mask 和设置，它可能选择 FlashAttention、memory-efficient kernel 或 math fallback。调用 SDPA 不代表每次都实际运行某个特定 FA 版本，应通过 profiler 和 backend 配置确认。

FlashAttention 与 FlashInfer

FlashInfer 是面向 LLM serving 的 kernel 库，覆盖 paged KV、prefill、decode、sampling 等操作。它可以包含受 FlashAttention 启发或兼容的 attention kernel，但它不是“FlashAttention 的后续论文版本”。

FlashAttention 与 PagedAttention

PagedAttention 主要定义 KV Cache 的分页布局和服务内存管理；FlashAttention 主要优化 attention 的 tile 数据流。生产引擎可以组合两者：KV 物理上分页，kernel 根据页表高效读取并执行 I/O-aware attention。

常见失败与排查思路

安装成功但没有加速

可能原因包括：

  • 输入 dtype/设备不受 kernel 支持；
  • head dimension、mask、dropout 或 GQA 形状触发 fallback；
  • 序列太短，launch 和调度开销占主导；
  • 实际瓶颈在 MLP、MoE、通信或 CPU；
  • PyTorch、CUDA、GPU 架构与 wheel 编译目标不匹配。

显存下降但墙钟时间变化不大

减少中间激活可以允许更大 batch，却不保证固定 batch 下显著变快。应分别测 kernel latency、端到端 step time、峰值显存，以及增大 batch 后的吞吐上限。

FA 版本数字越大就一定越好

FA3 针对 Hopper，FA4 针对 Blackwell。旧硬件可能根本不能运行相应路径；新实现也可能暂未覆盖某些 head dim、dropout、backward 或 paged KV 组合。正确选择依赖硬件和 feature matrix，而不是只看版本号。

如何验证一个 attention kernel

正确性

对小尺寸使用 FP32/高精度参考实现，比较 forward 和 backward：

torch.testing.assert_close(actual, expected, rtol=..., atol=...)

覆盖 causal/non-causal、不同序列长度、非整 tile 边界、GQA/MQA、极端 logits、padding 和不同 head dimension。半精度下误差阈值必须结合累积长度制定。

性能

记录：

  1. forward/backward 或 prefill/decode 分开测；
  2. batch、heads、KV heads、sequence length、head dim；
  3. dtype、GPU、CUDA、driver 和 kernel commit；
  4. warmup、CUDA Event 同步方式和重复次数；
  5. TFLOPS、GB/s、峰值显存和端到端 tokens/s；
  6. 是否发生 fallback。

只测一个 N=4096 形状无法说明长上下文、短 decode 或训练场景的整体表现。

总结

FlashAttention 最核心的思想可以浓缩为一句话：不要把昂贵的中间矩阵写到远处，再花更大代价读回来。 通过 online softmax，attention 可以在 tile 级别维护最大值、归一化因子和输出累积量，从而把完整分数矩阵留在“从未物化”的状态。

FA1 建立了 I/O-aware exact attention；FA2 重新安排 block 和 warp；FA3 把 Hopper 的 TMA/WGMMA 组织成异步流水；FA4 面向 Blackwell 重新平衡 Tensor Core、TMEM、shared memory 与 softmax。算法演进表面上是版本升级，本质上是不断追问：在这一代硬件上，数据和计算究竟在哪里等待。

因此，FlashAttention 的正确理解不是“把 \(O(N^2)\) 变成 \(O(N)\)”，而是“用线性辅助内存和更少 HBM I/O，高效完成原本的平方级精确 attention”。
