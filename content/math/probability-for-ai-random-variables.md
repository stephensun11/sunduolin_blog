## 为什么 AI 离不开概率

很多 AI 问题本质上都不是“确定答案”，而是在不确定性下做判断。

模型输出的不是一句话的绝对真理，而是下一个 token 的概率分布；分类模型输出的不是“它一定是猫”，而是每个类别的置信度；强化学习里，策略也常常是动作上的概率分布。

所以概率论不是一门离 AI 很远的数学课。它更像是理解模型输出、训练目标和评估指标的底层语言。

## 随机变量

随机变量可以理解为“随机实验结果的数值化表示”。

比如抛硬币可以定义：

```text
X = 1 表示正面
X = 0 表示反面
```

在 AI 里，随机变量更常见的形式包括：

- 一个样本的类别标签。
- 一个 token 的下一个词。
- 一个模型预测是否正确。
- 一个强化学习动作带来的回报。

随机变量让我们可以把现实中的不确定事件转成数学对象。

## 概率分布

概率分布描述随机变量取不同值的可能性。

对于离散随机变量，可以写成：

```text
P(X = x)
```

例如一个三分类模型输出：

| 类别 | 概率 |
| --- | --- |
| 猫 | 0.70 |
| 狗 | 0.20 |
| 鸟 | 0.10 |

这就是一个分布。概率总和为 1，每个概率都在 0 到 1 之间。

语言模型也是类似的。给定上下文后，模型会对整个词表给出一个概率分布，然后再通过贪心、采样、top-k、top-p 等方法选择下一个 token。

## 期望

期望是随机变量的平均趋势。

离散情况下：

```text
E[X] = sum_x x * P(X = x)
```

直觉上，期望不是某一次一定会发生的结果，而是重复很多次后的平均值。

在机器学习里，损失函数常常写成期望形式：

```text
E[loss(model(x), y)]
```

意思是我们希望模型在数据分布上的平均损失尽可能小。训练时拿到的是有限样本，所以会用 mini-batch 的平均损失来近似这个期望。

## 方差

方差衡量随机变量围绕期望的波动程度：

```text
Var(X) = E[(X - E[X])^2]
```

方差越大，结果越不稳定。

在 AI 里，方差经常出现在这些讨论中：

- 训练 loss 波动很大，可能说明 batch 太小或学习率太高。
- 强化学习回报方差很大，训练会变得不稳定。
- 采样温度越高，生成结果通常越发散。

理解方差之后，再看“降低估计方差”“稳定训练”这类说法，会具体很多。

## 条件概率

条件概率表示在已知某个事件发生的前提下，另一个事件发生的概率：

```text
P(A | B)
```

语言模型的训练目标就可以看成条件概率建模：

```text
P(next token | previous tokens)
```

模型每次生成一个 token，都是在当前上下文条件下预测下一个 token 的分布。

## 小结

随机变量把不确定事件变成数学对象，概率分布描述可能性，期望描述平均趋势，方差描述波动，条件概率描述上下文依赖。

这些概念合在一起，就是理解分类、生成、采样、损失函数和强化学习的基础。
