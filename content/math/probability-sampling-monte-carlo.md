## 为什么机器学习离不开采样

机器学习理论里经常写期望：

```text
E[loss(f(x), y)]
```

但真实训练时，我们拿不到完整的数据分布，只能拿到有限样本。于是训练会用样本平均去近似期望：

```text
1/n * sum_i loss(f(x_i), y_i)
```

这就是经验风险最小化的基本想法。

采样把“对整个分布求平均”变成“对抽到的样本求平均”。

## 蒙特卡罗估计

蒙特卡罗方法的核心很朴素：如果一个量可以写成期望，就可以用随机样本的平均来近似。

假设：

```text
mu = E[g(X)]
```

从分布中采样：

```text
x1, x2, ..., xn
```

估计量是：

```text
mu_hat = 1/n * sum_i g(x_i)
```

样本越多，估计通常越稳定。

## mini-batch 训练就是一种估计

神经网络训练时，全量数据上的梯度可能很贵。实际训练通常抽一个 mini-batch：

```text
gradient_batch = average gradient on batch
```

这不是完整梯度，而是完整梯度的随机估计。

batch 越小，估计噪声通常越大；batch 越大，估计更稳定，但计算和显存成本也更高。

所以 batch size 不只是工程参数，它也影响优化里的随机性。

## 估计方差

采样估计会有波动。这个波动可以用方差理解。

如果估计量方差很大，就会出现：

- 训练 loss 抖动明显。
- 强化学习回报估计不稳定。
- 生成采样结果差异很大。
- 评估指标在小测试集上波动明显。

降低方差的方法包括增加样本数量、使用更稳定的估计器、做 baseline、控制采样温度等。

## 大数定律的直觉

大数定律告诉我们，独立同分布样本的平均值会随着样本数量增加而接近期望。

对机器学习来说，这解释了为什么“更多数据”通常能让估计更靠谱。

但它不保证小样本一定稳定，也不保证训练集分布和真实线上分布一致。样本平均只是在当前采样机制下接近期望。

如果采样有偏，估计也会有偏。

## 重要性采样的直觉

有时目标分布不好采样，但另一个分布更容易采样。重要性采样会从容易采样的分布里取样，再用权重校正。

直觉上，它在说：如果某些样本不是从目标分布自然来的，就要按概率比重新加权。

推荐系统、强化学习离线评估、稀有事件估计里都可能遇到类似思想。

这一块不需要一开始就深挖公式，但要知道采样分布和目标分布不一致时，直接平均可能会出问题。

## 生成模型里的采样

语言模型推理时也在采样。

模型给出下一个 token 的概率分布后，可以用不同策略选 token：

- greedy：永远选概率最大的 token。
- temperature sampling：调整分布尖锐程度。
- top-k：只在概率最高的 k 个 token 中采样。
- top-p：只在累计概率达到阈值的候选集合中采样。

这些策略都在控制随机性。随机性更强，结果更丰富，但也更容易跑偏。

## 小结

采样是机器学习把概率理论落到计算里的桥。样本平均近似期望，mini-batch 梯度近似全量梯度，生成模型通过采样把概率分布变成具体输出。

理解采样，就能更好地理解训练波动、评估置信度和生成随机性。
